# 基金快看（JK_1 + JJKK_2）完整项目文档

## 一、项目概述

"基金快看"是一个面向基金投资场景的微信小程序，帮助用户实时查看大盘行情、管理自选基金、模拟交易持仓、分析盈亏、阅读市场资讯。

项目由两个独立仓库组成：

| 仓库 | 路径 | 说明 |
|------|------|------|
| JK_1 | F:\JJGG\JK_1 | 前端，基于 uni-app x (Uvue) 的微信小程序 |
| JJKK_2 | F:\JJGG\JJKK_2 | 后端，基于 Spring Boot 3 的 Java 服务 |

核心能力：
- 大盘实时行情：覆盖 A 股、港股、美股、黄金等 14 个核心指数
- 基金估值追踪：分钟级估算净值，收盘后自动校正为官方净值
- 自选基金管理：搜索、添加、排序、删除，支持官方基金库同步
- 模拟交易系统：买入/卖出、多账户、交易流水、持仓盈亏
- 盈亏分析：日内分时、收益走势、盈亏日历、明细拆解
- 市场资讯：实时快讯、重要标记、详情阅读
- 价格提醒：自定义涨跌幅/净值目标，多种提醒时间模式
- 弹幕互动：基金分时图上叠加用户弹幕
- 深色模式：全局明暗主题切换

## 二、技术栈

### 前端（JK_1）
- 框架：uni-app x，使用 Uvue 语法（.uvue 文件）
- 语言：JavaScript + UTS（uni-app x 的类型语言）
- 构建目标：微信小程序（mp-weixin）
- 样式：纯 CSS（无预处理器），全局样式定义在 App.uvue
- 图表：自研 DOM 线段式分时图（line-chart.js），不依赖第三方图表库
- 网络：uni.request（REST）+ uni.connectSocket（WebSocket）

### 后端（JJKK_2）
- 语言：Java 17
- 框架：Spring Boot 3.5.14
- 数据库：MySQL 8（库名 fund_quick_look_jk）
- 缓存：Redis（分时数据缓存 + 认证 Token + 会话管理）
- 实时推送：Spring WebSocket
- ORM：Spring JDBC + 手写 RowMapper（无 MyBatis/JPA）
- 定时任务：Spring Scheduler
- 文件处理：Apache POI（Excel 解析）
- 构建工具：Maven

### 基础设施
- 后端默认端口：8080，上下文路径 /api
- 数据库连接：MySQL localhost:3306/fund_quick_look_jk
- Redis：localhost:6379

### 4.2 分时图引擎（common/line-chart.js）

这是整个前端最核心的技术模块，完全用纯 JS 实现，不依赖任何图表库。

**设计思路：**
- 用 DOM 绝对定位 + CSS transform:rotate 画线段
- 每两个相邻数据点生成一个 line-segment div
- 线段宽度 = 两点距离 + 1.5px（防亚像素缝隙）
- 线段角度 = atan2(dy, dx)（旋转定位）
- 渐变背景用 linear-gradient CSS 实现

**关键函数：**

1. `normalizeLinePoints(rawPoints, options)` - 数据归一化
   - 过滤无效值
   - 按交易时间排序（tradingMinuteIndex）
   - 去重（同一分钟只保留最后一条）

2. `buildLineChartModel(rawPoints, options)` - 生成图表模型
   - 计算坐标系（chartArea）
   - 对称/非对称 Y 轴（symmetric 用于分时，非对称用于历史）
   - 生成线段数组 segments[]
   - 零线、中线、活动线位置
   - 渐变背景 CSS

3. `locateLinePoint(rawPoints, touchX, width, options)` - 触摸交互定位
   - 将触摸 X 坐标映射到最近数据点
   - 支持交易时间轴定位（tradingMinuteIndex）

4. `buildLineMarkerViews(rawPoints, rawMarkers, options)` - 交易标记
   - 在图表上标记买卖点（黄色圆点=买入，蓝色圆点=卖出）

**交易时间轴系统：**

支持 7 种市场的交易时间映射：

| 市场 | 交易时段 | 分钟总数 |
|------|---------|---------|
| A股（SSE/SZSE） | 09:30-11:30, 13:00-15:00 | 241 |
| 港股（HKEX） | 09:30-12:00, 13:00-16:00 | 331 |
| 美股（NYSE/NASDAQ/AMEX） | 09:30-16:00 | 391 |
| 上期所（SHFE） | 21:00-02:30, 09:00-11:30, 13:30-15:00 | 572 |
| 上金所（AU9999） | 20:00-02:30, 09:00-11:30, 13:30-15:30 | 662 |
| 外汇/其他 | 06:00-05:00（次日） | 连续 |

这使得不同市场的分时图 X 轴能正确对齐交易时段，午休时段在图表上自然跳过。

## 五、前端页面详解

### 5.1 大盘页（pages/index）

**功能：** 展示全球主要指数实时行情，按市场分组。

**数据分组：**
- A 股：上证指数、深证成指、创业板指、沪深300、中证500、科创50
- 港股：恒生指数、恒生国企、恒生科技
- 美股：道琼斯、标普500、纳斯达克100
- 黄金：纽约黄金、上金Au99.99

**交互：**
- 顶部 Hero 区域显示上涨/下跌数量、行情更新时间、本地刷新时间
- 刷新按钮触发 getMarketOverview()
- 点击卡片跳转到 market-detail 详情页
- 卡片背景色按涨跌变化（红涨、绿跌、白平）

**技术细节：**
- DEFAULT_INDICES 定义 14 个指数的默认配置（市场+代码+名称）
- mergeDefaultIndices 将后端返回的实时数据合并到默认配置
- computed 属性 marketGroups 按市场自动分组

### 5.2 自选页（pages/watchlist）

**功能：** 用户自选基金列表，支持搜索添加、排序编辑。

**顶部 Hero：** 基金数量、平均估算涨幅、上涨/下跌数

**搜索功能：**
- 底部弹出搜索面板（popup-mask）
- 先搜索本地基金库（searchFunds）
- 本地无结果时搜索官方基金库（searchOfficialFunds）
- 搜索结果可直接添加自选

**列表功能：**
- 显示基金名称、代码、估算涨幅、最新净值
- 估算状态标记（实时估算/已收盘/估算延迟/估算异常）
- 支持按估算涨幅升序/降序排序
- 编辑模式：上移/下移调整顺序、删除自选
- 编辑完成后批量保存排序（reorderWatchlist）

**免责声明：** 底部固定展示，说明估算数据仅供参考。

### 5.3 持有页（pages/holdings）

**功能：** 模拟基金交易，管理持仓和交易记录。

**顶部 Hero：**
- 总成本、总市值、总盈亏额（可点击跳转盈亏分析）、总盈亏率
- 持有数量统计、更新时间

**操作区：**
- 模拟买入/模拟卖出按钮
- 账户切换条（横向滚动，支持多账户）

**持有明细：**
- 每只基金显示：名称、代码、账户、状态（已更新/等待中）
- 指标：持有份额、持仓成本、估算市值、盈亏率、盈亏额
- 点击基金可快速填充交易表单
- 支持按盈亏率排序

**交易记录：**
- 最近 30 条模拟成交
- 买入/卖出标签、基金信息、金额、净值、份额

**模拟交易弹窗：**
- 买入/卖出切换
- 基金代码（6位）、账户名称、成交金额、成交份额
- 成交净值（可空，自动按交易时段确认净值日）
- 手续费
- 提交后如果净值未确认，状态为"等待中"

### 5.4 信息页（pages/information）

**功能：** 展示近三天市场资讯。

**功能：**
- 顶部筛选：全部/重要切换
- 卡片列表：发布时间、标题、摘要、预览图
- 重要资讯红色标记
- 分页加载（每页6条，上滑加载更多5条）
- 点击跳转资讯详情

### 5.5 我的页（pages/me）

**功能：** 用户个人信息、设置、反馈。

**个人资料区：**
- 头像（支持微信头像选择上传）
- 昵称（支持微信昵称 type="nickname"）
- 注册天数、注册日期、最近登录

**菜单项：**
- 盈亏分析（跳转）
- 显示设置（浅色/深色模式切换）
- 建议反馈（分类+内容+联系方式，提交到后端日志）
- 账号信息（弹窗显示用户号等）
- 免责声明（弹窗）
- 重新登录（清除登录态，重新微信登录）

**首次引导：** 如果用户未设置头像或使用默认昵称，自动弹出资料完善面板。

### 5.6 基金详情页（pages/fund-detail）

**功能：** 单只基金的完整信息和交互。

**顶部 Hero：**
- 返回按钮、基金名称、代码
- 数据状态标记（实时估算/已更新/估算延迟/估算异常）

**统计卡片：**
- 估算涨幅/真实涨幅（自动切换）
- 日涨幅、最新净值

**分时图表：**
- 今天/近1月/近3月/近6月/近1年 五个时间范围
- 触摸交互：手指滑动查看任意时间点数值
- 交易标记：历史买卖点在图表上用圆点标记
- 零线（虚线）：对称显示涨跌

**弹幕系统（仅今天分时）：**
- 弹幕叠加在分时图上方
- 6 种颜色可选
- 最多 15 条，10 分钟后自动消失
- 支持开启/屏蔽弹幕

**操作区：**
- 添加持有（快速买入弹窗）
- 删除自选
- 提醒设置（弹窗）

**提醒设置弹窗：**
- 四种条件：日涨幅、日跌幅、涨到目标净值、跌到目标净值
- 三种提醒时间：及时提醒、14:30 统一提醒、自定义时间
- 每个条件独立开关

**数据刷新：** 分时模式每 60 秒自动刷新

### 5.7 大盘详情页（pages/market-detail）

**功能：** 单个指数的分时/历史走势。

与基金详情页结构类似，但：
- 数据源为 marketMinute / marketHistory
- 显示点位而非净值
- 根据市场类型自动选择 X 轴标签
- 分时模式每 30 秒自动刷新

### 5.8 盈亏分析页（pages/profit-analysis）

**功能：** 全面的投资收益分析工具。

**顶部区域：**
- 账户切换（横向滚动）
- 今日盈亏汇总：金额、收益率、当前市值、昨日市值、更新时间

**收益走势图：**
- 五个时间范围：当日/本周/本月/本年/全部
- 两种指标切换：金额/收益率
- 金额/收益率图表联动

**盈亏日历：**
- 三种视图：每日/每月/每年
- 日历单元格背景色按盈亏深浅变化
- 选中日期后下方明细同步更新
- 支持月份/年份前后翻页

**盈亏明细：**
- Treemap 风格色块图（红盈绿亏）
- 列表展示：基金名称、代码、账户、占比、盈亏金额、盈亏率
- 顶部色条颜色深度表示贡献占比

**自动刷新：** 每 60 秒自动更新

### 5.9 板块涨跌页（pages/sector-rankings）

**功能：** 展示全部板块按涨跌幅排行。

- 排名序号、板块名称、代码、涨跌幅、最新点位
- 红涨绿跌颜色标记
- 最多展示 100 个板块

### 5.10 基金搜索页（pages/funds）

**功能：** 基金搜索和官方同步工具。

- 本地库模式：搜索已入库基金
- 官方同步模式：从证监会产品索引拉取并写入本地库
- 搜索结果显示基金信息、净值、估算涨幅
- 复制基金代码功能

## 七、后端接口详解

### 7.1 认证接口

**POST /api/auth/wechat-login**
- 请求：{ code, nickname }
- 流程：调用微信 jscode2session → 查找/创建用户 → 生成 Token → 存入 Redis
- 响应：{ userId, token }
- Token 有效期：7 天（可配置）

**认证中间件：**
- 所有需要登录的接口通过 Authorization: Bearer <token> 头校验
- AuthService.requireUser 校验 Token 并提取 userId
- Token 存储在 Redis：jjkk:auth:token:{token} → AuthSession

### 7.2 市场行情接口

**GET /api/market/overview**
- 返回大盘概览：14 个指数的最新价格、涨跌幅、涨跌额、更新时间
- 数据源优先级：东方财富 → 新浪 → 腾讯 → Yahoo → MySQL 缓存
- 美元指数被过滤，不返回给前端

**GET /api/market/indices/{symbol}/minute**
- 返回指数当日分时数据（分钟级）
- 优先读 Redis，无数据时读 MySQL，最后尝试最新行情兜底
- 交易时间内每 30 秒自动刷新

**GET /api/market/indices/{symbol}/history?range=1m|3m|6m|1y**
- 返回指数历史走势（日K）
- 优先用东方财富 K 线，回落到新浪/Yahoo/MySQL

**GET /api/market/sector-rankings?limit=10**
- 返回板块涨跌排行
- 从东方财富板块行情接口获取

**GET /api/market/fund-rankings?limit=10**
- 返回基金排行

### 7.3 基金数据接口

**GET /api/funds?keyword=xxx&page=1&size=20**
- 搜索本地基金库（fund_share_class 表）
- 支持基金代码、名称、拼音搜索

**GET /api/funds/official/search?keyword=xxx&limit=20**
- 从东方财富基金搜索接口获取
- 自动写入本地 fund_product + fund_share_class
- 返回结果含本地已同步的基金

**GET /api/funds/{fundCode}**
- 基金详情：基础信息 + 最新净值 + 估算数据
- 访问时会触发净值同步和收盘校正

**GET /api/funds/{fundCode}/minute**
- 基金当日分时估算数据
- 优先 Redis → MySQL → 最新快照兜底

**GET /api/funds/{fundCode}/history?range=1m|3m|6m|1y**
- 基金历史净值走势
- 从东方财富同步净值历史

**GET /api/funds/{fundCode}/bullet-comments**
- 获取基金弹幕列表

**POST /api/funds/{fundCode}/bullet-comments**
- 发送弹幕
- 请求：{ content, color }

### 7.4 用户自选接口

**GET /api/users/{userId}/watchlist**
- 联表查询 fund_nav_daily + fund_realtime_snapshot
- 返回估算涨幅、估算净值、最新官方净值

**POST /api/users/{userId}/watchlist**
- 添加自选，检查基金是否存在
- 添加后立即触发估值刷新

**PUT /api/users/{userId}/watchlist/order**
- 批量更新排序

**DELETE /api/users/{userId}/watchlist/{watchId}**
- 删除自选

### 7.5 持仓交易接口

**GET /api/users/{userId}/holdings/dashboard?accountId=&txnLimit=30&sortBy=pnlPct&sortDirection=desc**
- 一次性返回：汇总数据 + 账户列表 + 持有明细 + 交易记录
- 持有明细含估算市值、估算盈亏
- 交易记录含等待确认状态

**POST /api/users/{userId}/holdings/trades**
- 模拟买入/卖出
- 请求：{ fundCode, txnType, accountName, amount, share, nav, fee }
- 如果净值为空，自动按交易时段确认净值日
- 交易时段（09:30-15:00）和午休（11:30-13:00）按今天净值
- 其他时间按下一个交易日净值
- 状态为 PENDING 时等待官方净值确认

**GET /api/users/{userId}/holdings/summary**
- 持仓汇总数据

**GET /api/users/{userId}/holdings/transactions?limit=30**
- 交易流水记录

### 7.6 盈亏分析接口

**GET /api/users/{userId}/profit-analysis?accountId=&range=&calendarMode=&anchor=&detailKey=**
- 返回复杂结构：
  - accounts: 账户列表

  - intraday: 日内分时盈亏点
  - trend: 区间收益走势

  - details: 选中日期的盈亏明细
- 计算逻辑：基于持有份额 ×（估算净值 - 前一日净值）

### 7.7 提醒规则接口

**PUT /api/users/{userId}/fund-alerts/{fundCode}**

- 规则类型：日涨幅/日跌幅/涨到净值/跌到净值
- 提醒时间：及时/14:30/自定义

### 7.8 资讯接口

**GET /api/information?page=1&size=6&importantOnly=false**
- 分页返回资讯列表
- 重要关键词：基金、ETF、港股、美股、黄金、央行、政策、暴跌等

**GET /api/information/{id}**
- 资讯详情

### 7.9 用户接口

**GET /api/users/{userId}**
- 用户资料：昵称、头像、注册天数、显示ID

**PUT /api/users/{userId}/profile**

## 九、核心业务流程

### 9.1 微信登录流程

```
前端 uni.login 获取 code
  → POST /api/auth/wechat-login { code, nickname }
  → 后端调用微信 jscode2session 换取 openid
  → 查找 app_user 表，不存在则创建
  → 生成随机 Token，存入 Redis（TTL 7天）
  → 返回 { userId, token }
  → 前端存储 jk_user_id 和 jk_auth_token
```

### 9.2 基金估值刷新流程

```
每分钟定时任务触发
  → 查询活跃基金代码（自选+持有+热门）
  → 并发刷新每只基金（parallelism=6, batch=180）
  → 调用东方财富 fundgz 接口获取估算净值
  → 写入 fund_realtime_snapshot（MySQL）
  → 写入 Redis 分时点
  → WebSocket 推送给在线用户
```

### 9.3 收盘校正流程

```
每天 19:00-23:00 每半小时执行
  → 查询 data_status != CLOSED 的基金
  → 从东方财富同步当日官方净值
  → 用官方净值替换估算值
  → 更新 fund_realtime_snapshot 和 fund_estimate_minute
  → 标记 data_status = CLOSED
```

### 9.4 Redis 分时数据持久化流程

```
每天 00:00 执行
  → 从 Redis 集合获取前一交易日的基金/指数
  → 读取 hash 中所有分钟点
  → 批量写入 fund_estimate_minute / market_quote_minute
  → ON DUPLICATE KEY UPDATE 避免重复
```

### 9.5 模拟交易流程

```
用户提交买入/卖出
  → 校验基金代码（6位数字）
  → 校验金额或份额至少填一项
  → 如果净值为空，按规则确定净值日：
    - 交易时段 09:30-15:00 → 今天
    - 午休 11:30-13:00 → 今天
    - 其他时间 → 下一个交易日
  → 查询对应日期的官方净值
  → 如果净值已存在 → 立即成交，更新持仓
  → 如果净值未出 → 状态 PENDING，等待收盘校正
  → 写入 user_holding_txn 流水
```

## 十三、设计决策与注意事项

### 13.1 数据真实性

- 官方数据和估算数据严格分离，不互相覆盖
- 前端在未获得官方净值时显示"估算涨幅"，获得后显示"真实涨幅"
- 所有外部数据绑定 data_source 和 data_import_batch，保证可追溯
- 估算数据标注 confidence_score 置信度

### 13.2 分时图设计

- 自研 DOM 线段式图表，不依赖 ECharts/Chart.js
- 用 CSS transform:rotate 实现折线，性能优于 Canvas
- 支持 7 种市场交易时间轴，午休时段自动跳过
- 触摸交互通过 tradingMinuteIndex 精确定位

### 13.3 2核2G 优化

- 估算刷新设并发上限（parallelism=6）
- WebSocket 只推用户自选/持有基金
- Redis 分时数据 42 小时 TTL，不永久保存全市场
- 采集任务和 API 用同一进程，但线程池隔离

### 13.4 安全措施

- 用户只能访问自己的数据（userId + token 校验）
- 敏感配置通过环境变量注入
- 上传头像限制文件大小和类型
- 反馈日志脱敏处理

## 十四、后续扩展建议

1. **数据源解耦：** 把第三方数据源访问抽成独立 source 包，便于管理限流和熔断
2. **估算模型版本化：** 支持对比不同算法准确率
3. **WebSocket 精准推送：** 建立用户订阅表，只推自选基金变更
4. **资讯去重：** 增加去重指纹，避免不同源重复快讯
5. **管理后台：** 查看数据源健康度、失败次数、估值覆盖率
6. **监控指标：** Prometheus 监控第三方请求耗时、Redis 命中率
7. **基金弹幕增强：** 弹幕持久化、热门弹幕排行
8. **多平台支持：** App.uvue 已预留 H5/App 配置，可扩展到其他平台
